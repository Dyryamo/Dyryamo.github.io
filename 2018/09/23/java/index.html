  4. getRequestDispatcher().forward();请求转发的方式跳转页面 A->B
  5. getServerContext() 或请项目的ServletContext对象

get 和 post

http://localhost:8888/JspProject/show.jsp?uname=sdf&name=sdf&pwd=sdf&hobby=%E8%B6%B3%E7%90%83
get 提交方式 method=“get” 和 地址栏 请求方式默认都属于get提交方式，form 表带默认是get模式

get 和 post 区别

  1. get方式在地址栏显示请求信息，但是地址栏能够容纳信息有限，4-5kb，且不安全
  2. 文件上传方式必须是post
  3. 推荐使用post

b. 修改server.xml 将tomcat 默认编码改为 utf -8 ，添加URIEncoding=”UTF-8”

页面跳转的两种方式

&nbps 请求转发 重定向
地址栏是否改变 不变 改变
是否保留第一次请求时的数据 保留 不保留
请求的次数 1 2
跳转位置 服务端 客户端发出的第二次request

重定向和转发打两种工作方式

转发（客户端） --> [ 服务窗口 A(服务端) -> 服务窗口B]// 服务器累不转发，实现页面跳转，所以请求1次，
重定向 (客户端) -->[服务窗口A] // A告诉客户端，让客户端发情球给B
        客户端---> 根据窗口A的回复，找B，于是客户端给B发request

jDBC(java database connectivity)

  1. JDBC API 提供各种操作访问的借口
  2. JDBC DriverManager : 管理不同的数据库驱动
  3. 各种数据库驱动，相应的数据库厂商提供的，直接操作数据库

JDBC 主要功能

三件事。具体通过一下接口，类实现
DriverManager 管理jdbc 驱动
connection 连接
statement 增删改查
callableStatement 调用数据库中的存储过程和存储函数
result 返回结果集

jdbc 访问数据库的具体步骤

a. 导入驱动，加载具体驱动类
b. 与数据库建立链接
c. 发送sql 执行
d. 处理结果集

数据库驱动

驱动jar 具体的驱动类 链接字符串
oracle ojdbc-x.jar oracle.jdbc.oracleDriver jdbc:oracle:thin:@localhost:1521:ORCL
mysql mysql-connector-java-x.jar com.mysql.jdbc.Driver jdbc:mysql://localhost:3306/数据库实例名
SqlServer sqljdbc-x.jar com.microsoft.sqlserver.jdbc.sqlserverdriver jdbc:microsoft:sqlserver:localhost:1433;datebasename=实例名

javaBean

javaBean是一个java类的定义，满足以下两点可称为javaBean

a.public 修饰类，public 无参构造函数
b.所有的属性都死private，并且提供set/get/isd
